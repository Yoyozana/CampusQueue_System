## Risk Management Plan

- **Risk:** Delays in task completion  
  **Mitigation:** Weekly meetings and progress tracking  

- **Risk:** Communication breakdown among team members  
  **Mitigation:** Use group communication tools (WhatsApp / Teams) and regular updates  

- **Risk:** Technical difficulties with backend or database  
  **Mitigation:** Research, peer support, and incremental development, AI 

- **Risk:** Integration issues between frontend and backend  
  **Mitigation:** Test APIs early and frequently  

- **Risk:** Version control conflicts on GitHub  
  **Mitigation:** Use proper branching and pull requests  


## Technology Stack

- **Frontend:** HTML, CSS, JavaScript  
  *Reason:* Simple, widely supported, and suitable for building responsive user interfaces without heavy frameworks at initial stage  

- **Backend:** Python (Flask)  
  *Reason:* Lightweight framework, easy to learn, and efficient for building REST APIs  

- **Database:** SQL (MySQL / SQLite)  
  *Reason:* Structured data storage suitable for managing users, queues, and tickets  

- **Version Control:** Git & GitHub  
  *Reason:* Enables collaboration, version tracking, and code management
