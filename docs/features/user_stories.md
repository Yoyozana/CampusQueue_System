## Student User Stories
As a student, I want to register and log in so that I can access the queue management features and track my history.
As a student, I want to join a queue online so that I don't have to wait in a physical line and can spend my time more productively.
As a student, I want to receive a digital queue ticket number so that I have a reference for my position in line.
As a student, I want to view my current position in the queue so that I can estimate when it will be my turn.
As a student, I want to receive real-time notifications so that I am alerted immediately when I am next in line to be served.

## Admin & Staff User Stories
As an admin, I want to view all active queues so that I can monitor the current demand and workload across the campus.
As a service point staff member, I want to call the next student in the queue so that I can efficiently manage the flow of traffic.
As an admin, I want to manage service points (add, edit, or remove) so that the system accurately reflects the available service locations on campus.
As an admin, I want to monitor queue activity and metrics so that I can identify bottlenecks and improve overall campus efficiency.

## Technical Implementation Tasks (for the Team)
Front-end: Build responsive interfaces using JavaScript, CSS, and HTML for both students and admins.
Back-end: Develop a robust API using Python to handle queue logic and user authentication.
Database: Design and implement a MySQLschema to store user data, queue status, and service point details.
Deployment: Set up a continuous deployment pipeline to host the front-end application on Netlify
